## **Introduction**

This document describes the process by which external datasets are approved for incorporation into Google’s Data Commons knowledge graph (the “DC graph”). Our process ensures that every dataset incorporated into our [DataCommons.org](http://DataCommons.org) instance falls under [CC-BY 4.0](https://creativecommons.org/licenses/by/4.0/deed.en), equivalent, or more permissive licensing terms. Maintaining a formalized process to validate and approve licensing terms for the datasets that underpin the DC graph is a crucial component of the Data Commons mission. By ensuring that all datasets that underpin the DC graph maintain appropriate terms for use, Data Commons can maximize its utility, both as a resource for internal partners and a tool for external users.

This process assumes internal data consumers have carefully selected the external candidate dataset(s) for integration with the DC graph; the process for vetting an external dataset’s content for risk and accuracy is not covered in this document.

## **Overview**

### **What is Data Commons?**

Data Commons synthesizes a single graph from these different data sources. It links references to the same entities (such as cities, counties, organizations, etc.) across different datasets to nodes on the graph, so that users can access data about a particular entity aggregated from different sources without data cleaning or joining. At its core, Data Commons is not so much a piece of infrastructure as it is a body of structured data, together with its ontology, schema, etc. 

### **Why a knowledge graph?**

A knowledge graph allows us to create a single data graph that is interconnected between many domains. It is also a flexible way to represent data about many things. The graph includes properties and statistics about many types of “[Things](https://datacommons.org/browser/Thing)”.

Data Commons is accessible to the public via the various visual tools and [APIs](https://docs.datacommons.org/api) available on [https://datacommons.org](https://datacommons.org/).

### **Why choose datasets released under CC BY 4.0**

The [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/deed.en) license allows users to freely share, modify, and distribute the work for both commercial and non-commercial purposes. The only requirement is that proper attribution is given to the original creator(s). This license is globally recognized and ensures the work can be used and built upon without restrictive legal barriers, making it an ideal license type to support the DC graph.

## **Data intake approval process**

Below outlines each of the options for a dataset to be approved for incorporation into the DC graph. Please note that ALL datasets in the DC graph must have a documented approval.

If you have questions about the process or need assistance, reach out to tmonty@google.com.

### **Option 1: License review**

Before using a dataset, please verify that the dataset is governed by a CC BY 4.0 or equivalent license. Please see a list of pre-approved licenses below:

| *Pre-approved license type*  |
| :---- |
| CC BY 4.0 / CC BY 1.0 |
| CC BY IGO |
| Data that’s in the public domain, as long as no further restrictions apply. If you have any questions, please contact your pcounsel |
| MIT License |
| Apache 2.0 |

If a dataset **is** governed by one of the pre-approved licenses, you may incorporate it into the DC graph.

If a dataset **is** **not** governed by any of the license types above, please use Options 2 or 3 to seek approval for use.

### **Option 2: Legal review**

If the dataset is not governed by a pre-approved license, please reach out to your pcounsel (tmonty@) for legal review. When submitting your review request, please include the following:

* A link to the data you wish to ingest  
  * *NOTE: Please include a link to the specific dataset. A general website link will not provide the level of specificity required to document our review. If you wish for Legal to review multiple datasets provided by the same dataowner, please link each of specific datasets for review.*

* A link or copy of the license or terms of service that govern the datasetset.

The timeframe of legal review will depend on the number of datasets contained in your review request. For \<10 requests, review usually takes 5 business days. For time sensitive requests, please include “PRIORITY” in your review request.

### **Option 3: Explicit authorization from dataset owner**

If a dataset does not have a governing set of licensing terms, or if you have been instructed to do so by your pcounsel, please seek approval by reaching out to the dataowner directly for explicit authorization to use their data. Please use this [email template](https://docs.google.com/document/d/1CrUbF_ahkjtp0iEfbAF0Fk81EW5FYqPZMLXUbInfICc/edit?tab=t.0) to reach out to the appropriate dataowner.  

Once authorization has been received, please save a copy of the authorization email from the dataowner for recordkeeping purposes.
